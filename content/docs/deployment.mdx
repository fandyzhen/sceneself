## Prerequisites

Before deploying, ensure you have:

- PostgreSQL database (Supabase, Neon, or Vercel Postgres)
- OpenRouter API key (required for Scene generation)
- Creem account and API key
- Resend API key
- (Optional) Google OAuth credentials
- (Optional) S3-compatible storage

## Deploy to Vercel

### 1. Push to GitHub

```bash
git push origin main
```

### 2. Import in Vercel

1. Go to [vercel.com](https://vercel.com)
2. Import your GitHub repository
3. Set all environment variables (see [Environment Variables](./environment))
4. Deploy

### 3. Database Migration

After first deploy:

```bash
pnpm db:push
```

### 4. Configure Webhook

In Creem Dashboard, set webhook URL:

```
https://your-domain.com/api/payments/creem/webhook
```

### 5. Set Up Cron Jobs

For subscription credit grants and selfie cleanup, configure cron jobs to call hourly:

```bash
curl -H "Authorization: Bearer YOUR_CRON_SECRET" \
  https://your-domain.com/api/cron/subscription-grants
curl -H "Authorization: Bearer YOUR_CRON_SECRET" \
  https://your-domain.com/api/cron/cleanup-selfies
```

On Vercel, add to `vercel.json`:

```json
{
  "crons": [
    { "path": "/api/cron/subscription-grants", "schedule": "0 * * * *" },
    { "path": "/api/cron/cleanup-selfies", "schedule": "0 * * * *" }
  ]
}
```

### 6. Create Admin Account

```bash
ADMIN_EMAIL=admin@example.com pnpm admin:setup
```

### 7. Verify Environment

Make sure production `.env` has correct values for:

- `BETTER_AUTH_URL` — Your actual domain
- `NEXT_PUBLIC_APP_URL` — Your actual domain
- `RESEND_FROM_EMAIL` — Verified domain sender
- `SCENE_DEV_FALLBACK="false"` — Disable placeholder images in production

## Post-Deploy Checklist

- [ ] Database migrated
- [ ] Webhook URL configured in Creem
- [ ] Cron jobs set up for credit grants and selfie cleanup
- [ ] Admin account created
- [ ] Email sender domain verified in Resend
- [ ] OAuth redirect URIs updated for production domain
- [ ] Analytics keys configured (PostHog, GA, Clarity)
