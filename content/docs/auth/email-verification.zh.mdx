## 工作原理

用户注册时会收到验证邮件。未验证也可以登录，但在验证完成之前，`EmailVerifiedGuard` 组件会限制保存、导出场景集等关键操作。

### 验证流程

1. 用户注册 → 自动发送验证邮件
2. 邮件包含带唯一 token 的验证链接
3. 用户点击链接 → 验证 token → 标记邮箱已验证
4. `EmailVerifiedGuard` 解锁被拦截的关键操作

### 重新发送验证

用户可以在 `/verify-email` 页面请求新的验证邮件。API 端点：

```
POST /api/auth/resend-verification
```

### 调整验证要求

默认登录不要求已验证邮箱（`lib/auth.ts` 中 `requireEmailVerification: false`）。如需强制验证后才能登录：

```typescript
// lib/auth.ts
emailAndPassword: {
  enabled: true,
  requireEmailVerification: true, // 强制验证后才能登录
}
```

如需完全取消对关键操作的拦截（不建议在生产环境使用），从 `app/[locale]/(protected)/layout.tsx` 中移除 `EmailVerifiedGuard`。
