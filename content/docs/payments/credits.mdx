## Architecture

The credit system uses a **dual-write pattern** for balance tracking:

- `user.credits` — Fast-access current balance (for quick reads)
- `creditLedger` — Immutable audit trail of all credit changes

Both are always updated in a single database transaction to ensure consistency.

## Core API

All functions are in `lib/credits.ts`:

```typescript
// Check user's balance
const credits = await getUserCredits(userId);

// Check if user can afford an operation
const canAfford = await canUserAfford(userId, 300);

// Deduct credits (transactional)
await deductCredits(userId, 300, 'scene_set');

// Refund credits (e.g., for undelivered photos)
await refundCredits(userId, 50, 'scene_refund');
```

## Credit Costs

| Operation | Cost | Configured In |
|-----------|------|---------------|
| Scene set (6 photos) | 6 × 50 = 300 credits | `lib/scene/pricing.ts` (`SHOTS_PER_SET`, `CREDITS_PER_PHOTO`) |

The full set price is deducted up front when the job starts; undelivered photos are refunded per photo when the job finishes.

## Credit Sources

| Source | Trigger | Amount |
|--------|---------|--------|
| Registration bonus | New user signup | 300 credits |
| Subscription | Webhook / cron | Per plan config |
| Redemption code | User redeems at `/redeem` | Per code config |
| Admin adjustment | Manual via admin panel | Custom |

## Ledger Reasons

Each `creditLedger` entry has a `reason` field:

- `registration_bonus` — Free credits on signup
- `scene_set` — Scene set deduction (300 credits)
- `scene_refund` — Refund for undelivered photos
- `subscription_cycle` — Subscription cycle grant
- `redemption` — Redemption code credits
- `admin_adjustment` — Manual admin change

## Credit Compensation

Scene generation uses a **deduct-first, refund-on-failure** pattern:

```typescript
// 1. Deduct the full set price when the job starts
await deductCredits(userId, 300, 'scene_set');

// 2. Run the generation job
// ...

// 3. Refund any undelivered photos when the job finishes
await refundCredits(userId, refundForUndelivered(delivered), 'scene_refund');
```

This ensures users never lose credits for photos that were not delivered.
